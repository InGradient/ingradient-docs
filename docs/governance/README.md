# Governance

## 이 폴더는 무엇인가
문서 체계, SSOT 기준, 프로젝트별 필수 문서, 문서 상태와 변경 절차를 정의한다.

## 언제 읽는가
- 새 서비스나 새 repo를 시작할 때
- 문서 위치와 역할을 정리할 때
- 기존 문서를 수정하거나 폐기하려 할 때

## 먼저 읽을 문서
- `project_documentation_requirements.md`: 프로젝트 유형별 필수 문서 기준
- `documentation_policy.md`: SSOT와 명명 규칙
- `document_lifecycle_policy.md`: Draft / Review / Approved / Deprecated 규칙

## 다음으로 읽을 문서
- 시스템 전체 기준은 `../architecture/README.md`
- 실제 repo 상세는 `../projects/README.md`
