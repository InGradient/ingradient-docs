# auth-service Upcoming

아직 릴리즈 버전이 정해지지 않은 아이디어, 개선사항, 기술 부채, 고객 요구사항의 대기열이다. 버전이 정해지면 `releases/v{버전}.md`로 승격한다.

## 새 기능 아이디어

| 우선순위 | 아이디어 | 배경 | 대상 버전 |
|----------|----------|------|-----------|
| 중간 | Organization 계층화 (상위 → 하위) | 같은 기업의 여러 팀을 팀 단위 Organization으로 운영할 때, "기업 전체"를 묶는 상위 Organization이 필요. 팀 단위 라이선스 판매 시 기업이 한번에 결제하고 팀별로 나누는 구조도 고려 | 미정 |
| 중간 | 팀 단위 라이선스/Seat 패키지 | 기업 단위가 아닌 팀(5명/10명/50명) 단위로 라이선스를 판매하는 모델. billing, subscription management, seat allocation 설계 필요. Organization 계층화와 연계 | 미정 |

## 개선사항

| 우선순위 | 항목 | 배경 | 대상 버전 |
|----------|------|------|-----------|
| 낮음 | 라이선스 명시적 연장 흐름 | 현재는 조직 라이선스 탭에서 만료일을 직접 수정해 연장. 명시적 "Renew" 버튼 → 연장 기간 입력 모달 → 새 만료일 계산 + 연장 이력 별도 기록하는 공식 운영 프로세스 도입. 감사 추적성 강화 | 미정 |
