# auth-service User Scenarios

- 문서명: auth-service User Scenarios
- 목적: 사용자가 특정 목적을 달성하기 위해 어떤 화면을 거치는지 흐름 단위로 정리한다.
- 상태: Draft
- Owner: Product / Auth
- 마지막 수정일: 2026-03-29

## 이 문서의 역할

- `product_guide.md`가 "각 화면에서 뭘 할 수 있는가"라면, 이 문서는 **"이 목적을 달성하려면 어떤 화면을 거치는가"**를 정리한다
- features/의 상세 시나리오(예외 흐름, 에러 처리, 테스트 계획)까지는 다루지 않는다
- 기술 용어 없이, 사용자 관점의 흐름으로 작성한다

## 시스템 초기 설정

### System Admin 첫 로그인 및 시스템 구성

1. 시스템 배포 시 생성된 기본 Admin 계정으로 Admin 대시보드에 로그인한다
2. Settings에서 시스템 기본 설정을 구성한다:
   - 보안 정책 기본값 (비밀번호 정책, MFA, 계정 잠금, 세션 제한)
   - 토큰/세션 설정 (Access Token TTL, Refresh Token TTL)
   - 시스템 설정 (시스템 이름/로고, 기본 역할, 초대 유효 기간)
   - 알림 설정 (이메일 발송 방식 구성 + 테스트 이메일 발송으로 확인)
   - 라이선스 기본값 (기본 플랜, maxUsers, maxDevices)
3. Product를 생성한다 (예: INGRADIENT Platform, MediLabel)
4. 첫 조직을 생성한다 (이름 입력 → 코드 자동 생성)
5. 조직에 라이선스를 할당한다 (사용자 수, 디바이스 수, 오프라인 허용 등)
6. 필요하면 조직 보안 설정에서 "커스텀 설정"으로 시스템 기본값을 오버라이드한다

## 사용자 온보딩

### Admin이 직접 사용자를 생성하는 경우

1. Admin이 사용자 관리 페이지에서 "+ 새 사용자"를 클릭한다
2. 모달에서 Login ID, 비밀번호, 비밀번호 확인, 이름, 이메일을 입력한다
3. 소속 조직을 선택한다 (선택 — 비워두면 소속 없는 개인 사용자)
4. Product를 선택한다 (다중 선택 가능) + Product별 역할 지정 — 조직 선택과 무관하게 Product 배정 가능
5. "생성"을 클릭한다 → Login ID 중복 검사 + 비밀번호 규칙 검증
6. 사용자가 Pending 상태로 생성된다
7. Admin이 사용자 목록에서 "승인"을 클릭하면 Active로 전환된다
8. 사용자가 Login ID + 비밀번호로 로그인한다. Product에 배정되어 있으면 해당 Product에 접근 가능하다. 조직 소속 여부와 무관하게 Product를 사용할 수 있으며, 이후 조직에 초대되거나 가입 코드로 합류할 수도 있다

### 이메일 초대로 가입하는 경우

1. Admin 또는 조직 관리자가 초대 관리 페이지에서 조직을 선택한다
2. "+ 새 초대"를 클릭한다 (조직 선택 후 활성화)
3. 모달에서 이메일, 역할, 유효 기간을 설정한다 (기본 유효 기간은 Settings에서 설정한 값)
4. "초대 발송"을 클릭한다
5. 초대 대상자에게 이메일이 발송된다 (Settings의 이메일 설정으로 발송)
6. 대상자가 초대 링크를 클릭하고 계정을 생성한다
7. 사용자가 Pending 상태로 생성되고 조직 멤버로 등록된다
8. Admin이 사용자를 승인하면 Active로 전환된다
9. 사용자가 로그인한다

### 가입 코드로 가입하는 경우

1. 조직 관리자가 가입 코드를 생성한다 (최대 사용 횟수, 만료일 설정 가능. 기본 만료는 Settings 값)
2. 가입 코드를 대상자에게 공유한다 (메신저, 게시판 등)
3. 대상자가 가입 코드를 입력하여 조직에 가입한다
4. 가입 코드에 설정된 기본 역할이 부여된다 (기본 역할은 Settings 값)

### SSO로 처음 로그인하는 경우

1. 조직 관리자가 조직 상세 > 보안 설정에서 SSO IdP를 등록한다 (Google, Microsoft 등)
2. JIT(자동 계정 생성)을 활성화하고 기본 역할을 지정한다
3. 사용자가 SSO 로그인 버튼을 클릭한다
4. 외부 IdP에서 인증 후 돌아온다
5. 계정이 자동 생성되고 조직 멤버로 등록된다

## 일상 사용

### 일반 로그인

1. 사용자가 Login ID + 비밀번호를 입력한다
2. MFA가 설정되어 있으면 TOTP 코드를 추가 입력한다
3. 로그인 성공 → 연동 제품(platform, edge 등)으로 이동

### 비밀번호를 잊었을 때 — 이메일로 직접 재설정

1. 로그인 화면에서 "비밀번호 찾기"를 클릭한다
2. 이메일을 입력한다
3. 재설정 링크가 이메일로 발송된다
4. 링크를 클릭하고 새 비밀번호를 설정한다 (비밀번호 규칙은 소속 조직 설정 또는 시스템 기본값 적용)
5. 새 비밀번호로 로그인한다

### 비밀번호를 잊었을 때 — Admin에게 초기화 요청

1. 사용자가 Admin에게 비밀번호 초기화를 요청한다
2. Admin이 사용자 상세 모달 > 기본 정보 탭에서 "비밀번호 초기화"를 클릭한다
3. 확인 다이얼로그 후 임시 비밀번호가 생성된다
4. 임시 비밀번호가 다이얼로그에 표시된다 (복사 버튼 포함)
5. Admin이 임시 비밀번호를 사용자에게 전달한다
6. 사용자가 임시 비밀번호로 로그인한다
7. 로그인 후 본인이 새 비밀번호로 변경한다

### 비밀번호를 N회 틀렸을 때

1. Settings 또는 조직 보안 설정에서 정한 횟수만큼 연속 로그인 실패한다
2. 계정이 설정된 시간 동안 잠금된다 (로그인 시도 불가)
3. 시간이 지나면 자동 해제되거나, Admin이 사용자 상세 모달에서 "잠금 해제"를 클릭한다

### 다른 디바이스에서 로그인한 것을 발견했을 때

1. 사용자 상세 모달 > 활동 탭에서 활성 세션 목록을 확인한다
2. 모르는 세션을 발견하면 해당 세션의 "종료" 버튼을 클릭한다
3. 또는 "전체 세션 종료"로 모든 세션을 한번에 종료한다
4. 비밀번호를 변경한다

### MFA를 설정할 때

1. MFA 설정을 시작한다
2. QR 코드가 표시된다
3. Google Authenticator 등 TOTP 앱으로 QR 코드를 스캔한다
4. 앱에 표시된 코드를 입력하여 설정을 완료한다
5. 백업 코드가 발급된다 (앱 분실 시 사용)

## 조직 운영

### 새 조직을 만들 때

1. Admin이 조직 관리 페이지에서 "+ 새 조직"을 클릭한다
2. 모달에서 조직 이름을 입력한다 → 코드가 자동 생성된다 (소문자 + 빈칸→`-`, 중복 시 `-2` 부여)
3. 필요하면 코드를 수동 수정한다
4. "생성"을 클릭한다
5. 조직 상세에서 라이선스를 할당한다
6. 보안 설정 탭에서 "시스템 기본값 사용" 또는 "커스텀 설정"을 선택한다
7. 커스텀 설정이면 이 조직만의 보안 정책을 설정한다

### 새 Product를 만들 때

1. Admin이 Product 관리 페이지에서 "+ 새 Product"를 클릭한다
2. 모달에서 Product 이름을 입력한다 → 코드가 자동 생성된다
3. 상태를 선택한다 (Active/Inactive)
4. "생성"을 클릭한다

### Product에 멤버를 배정할 때

1. Product 상세 모달 > 멤버 탭에서 "+ 멤버 추가"를 클릭한다
2. 사용자를 검색하여 선택한다
3. 역할을 지정한다
4. 멤버가 연동 제품에서 해당 Product에 접근할 수 있게 된다

또는:

1. 사용자 상세 모달 > 기본 정보 탭에서 "+ Product 추가"를 클릭한다
2. Product를 선택하고 역할을 지정한다

### 멤버 역할을 변경할 때

1. 조직 상세 > 멤버 탭에서 대상 멤버의 역할 드롭다운을 변경한다 → 즉시 저장
2. 또는 사용자 상세 모달 > 기본 정보 탭에서 Product별 역할 드롭다운을 변경한다

### 멤버를 제거할 때

1. 조직 상세 > 멤버 탭에서 대상 멤버의 제거 버튼을 클릭한다
2. 확인 다이얼로그 후 멤버가 제거된다
3. 해당 멤버의 모든 Product 접근도 함께 해제된다
4. 사용자 계정 자체는 삭제되지 않는다 (다른 조직에는 여전히 속할 수 있음)

## 디바이스 & 오프라인

### 현장에 Edge 디바이스를 배치할 때 (온라인)

1. Edge 디바이스에서 platform URL을 설정한다
2. 디바이스가 시스템에 연결하면 자동 등록된다 (상태: ACTIVE)
3. 라이선스 & 디바이스 페이지에서 등록된 디바이스를 확인한다

### 인터넷 없는 현장에 디바이스를 배치할 때 (오프라인)

1. Admin이 라이선스 & 디바이스 페이지에서 "+ 오프라인 라이선스 발급"을 클릭한다
2. 모달에서 하드웨어 fingerprint, 라벨, 조직, 만료일을 입력한다
3. "발급"을 클릭한다 → 라이선스 키가 다이얼로그에 표시된다 (복사 가능)
4. 라이선스 키를 복사하여 Edge 디바이스에 입력한다
5. 오프라인 엔타이틀먼트가 발급된다 (유효기간은 Settings의 기본값 또는 수동 설정)
6. 디바이스가 오프라인 상태에서 작업을 계속한다

### 온라인 디바이스를 폐기할 때

1. 라이선스 & 디바이스 페이지에서 대상 온라인 디바이스를 찾는다
2. "폐기"를 클릭한다 → 확인 다이얼로그
3. 해당 디바이스는 즉시 시스템에 접근할 수 없다 (상태: REVOKED)
4. 필요하면 폐기된 디바이스 레코드를 삭제한다

### 오프라인 디바이스를 폐기할 때

1. 현장 운영자가 Edge 디바이스에서 "Deactivate"를 누른다
2. 디바이스가 비활성화 코드를 생성하여 화면에 표시한다
3. 운영자가 비활성화 코드를 Admin에게 전달한다 (전화, 메신저 등)
4. Admin이 라이선스 & 디바이스 페이지에서 "오프라인 폐기"를 클릭한다
5. 모달에서 비활성화 코드를 입력하고 "폐기 처리"를 클릭한다
6. 디바이스 측은 Deactivate 시점에 이미 비활성화되어 있고, 서버 측에서도 폐기가 확인된다

## 보안 & 감사

### 시스템 보안 기본값을 변경할 때

1. Admin이 Settings > 보안 정책 탭에 진입한다
2. 비밀번호 정책, MFA, 계정 잠금, 세션 제한 등을 변경한다
3. "Save"를 클릭한다
4. "시스템 기본값 사용" 중인 모든 조직에 즉시 반영된다
5. "커스텀 설정" 중인 조직에는 영향 없다

### 조직별 보안 정책을 설정할 때

1. 조직 상세 > 보안 설정 탭에 진입한다
2. "시스템 기본값 사용" 상태면 현재 기본값이 읽기 전용으로 표시된다
3. "커스텀 설정"을 선택하면 모든 항목이 수정 가능해진다
4. 이 조직만의 보안 정책을 설정한다 (시스템 기본값보다 강하게 설정 가능)
5. "Save"를 클릭한다 → 해당 조직 멤버에게 즉시 적용

### 감사 로그를 확인할 때

1. Admin이 감사 로그 페이지에 진입한다
2. 상단 그래프에서 전체 현황을 한눈에 파악한다 (시간대별 추이, 이벤트 유형별 분포)
3. 이상이 보이면 해당 구간을 클릭하면 필터가 자동 적용된다
4. 하단 로그 목록에서 카테고리 → 이벤트 유형, 날짜 범위로 상세 필터링한다
5. 행을 클릭하면 이벤트 상세 정보가 드로어 또는 확장 행으로 표시된다
6. 필요하면 CSV 또는 JSON으로 내보낸다

### 의심스러운 활동을 발견했을 때

1. 감사 로그에서 반복 로그인 실패, 비정상 세션 등을 발견한다
2. 세션 관리 페이지에서 해당 사용자의 세션을 강제 종료한다 (이름 옆 세션 수 뱃지로 확인)
3. 또는 사용자 상세 모달 > 활동 탭에서 해당 사용자의 세션만 종료한다
4. 사용자 상세 모달 > 기본 정보 탭에서 "비활성화"를 클릭한다
5. 필요하면 "비밀번호 초기화"로 임시 비밀번호를 발급한다

### 알림 설정을 구성할 때

1. Admin이 Settings > 알림 탭에 진입한다
2. 이메일 발송 방식을 선택한다 (SMTP / SendGrid / SES)
3. 선택한 방식에 맞는 설정을 입력한다 (호스트, API Key 등)
4. 발신자 이메일과 이름을 입력한다
5. "테스트 이메일 발송"을 클릭하여 정상 동작을 확인한다
6. "Save"를 클릭한다
