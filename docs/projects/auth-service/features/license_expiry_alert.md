# 라이선스 만료 경고

- 문서명: 라이선스 만료 경고
- 목적: 라이선스 만료 임박 시 관리자에게 사전 알림을 보내 서비스 중단을 방지한다.
- 적용 범위: auth-service
- 상태: Draft
- Owner: Backend
- Reviewer: —
- 마지막 수정일: 2026-03-28
- 관련 SSOT 문서: gap-analysis.md

## 1. 기능 개요

- 기능 이름: 라이선스 만료 경고
- 한 줄 정의: 라이선스 만료 30/7/1일 전에 조직 관리자에게 이메일과 인앱 알림을 발송한다.

## 2. 문제 정의

- 라이선스가 예고 없이 만료되면 서비스 접근 차단으로 업무에 지장이 생긴다.
- 관리자가 수동으로 만료일을 확인해야 하는 불편함이 있다.

## 3. 목표

- 사용자 목표: 라이선스 만료 전 충분한 시간을 두고 갱신을 준비한다.
- 운영 목표: 만료 예정 조직을 자동으로 추적하고 알림을 발송한다.

## 4. 범위 / 비범위

- 포함: 만료 임박 알림 (30일/7일/1일), 이메일 + 웹훅 알림, 관리자 대시보드 경고 배너
- 비범위: 자동 갱신, 결제 시스템 연동

## 5. 사용자 시나리오

### 기본 흐름 — 만료 30일 전 알림

1. 참여자: 시스템 (스케줄러)
2. 선행 조건: 라이선스에 expiresAt이 설정됨
3. 기본 흐름: 일일 스케줄러 → 만료 30일/7일/1일 이내 라이선스 조회 → 조직 관리자에게 이메일 발송
4. 예외 흐름: 이미 알림 발송된 건 → 중복 방지 (알림 이력 확인)
5. 로그 포인트: `license_expiry_alert_sent`

## 6. UI / UX 방향

- Admin Console: 라이선스 목록에 만료 임박 배지 표시 (30일 이내 노란색, 7일 이내 빨간색)
- 조직 관리자 로그인 시 만료 임박 배너

## 7. 데이터 / API / 권한 영향

- 새 테이블: `alert_history` (org_id, alert_type, sent_at, channel)
- 스케줄러: cron job 또는 BullMQ 기반 일일 작업
- 이메일 템플릿: `license_expiry_warning` 추가
- 웹훅 이벤트: `license.expiry_warning`

## 8. 테스트 계획

- unit: 만료일 계산, 알림 대상 필터링, 중복 방지 로직
- integration: 스케줄러 실행 → 알림 발송 → 이력 기록
- e2e: 만료 30일 전 라이선스 설정 → 스케줄러 수동 트리거 → 이메일 수신

## 9. 릴리즈 고려사항

- release note: 라이선스 만료 사전 알림 기능 추가
- ops: 스케줄러 실행 환경 설정 (cron interval)

## 10. 오픈 이슈

- 알림 주기 커스터마이즈 여부
- 만료 후 유예 기간(grace period) 정책
