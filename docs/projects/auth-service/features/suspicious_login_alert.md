# 비정상 로그인 경고

- 문서명: 비정상 로그인 경고
- 목적: 새로운 IP나 디바이스에서의 로그인을 감지하여 사용자에게 알림을 보낸다.
- 적용 범위: auth-service
- 상태: Draft
- Owner: Backend
- Reviewer: —
- 마지막 수정일: 2026-03-28
- 관련 SSOT 문서: gap-analysis.md

## 1. 기능 개요

- 기능 이름: 비정상 로그인 경고
- 한 줄 정의: 사용자의 로그인 이력을 기반으로 새 IP/디바이스/지역 감지 시 이메일 알림을 발송한다.

## 2. 문제 정의

- 계정 탈취 시 사용자가 인지하기까지 시간이 걸린다.
- 비정상적인 로그인 패턴을 자동으로 감지하고 알려야 한다.

## 3. 목표

- 사용자 목표: 본인이 아닌 로그인 시도를 즉시 인지하고 대응한다.
- 운영 목표: 의심 로그인 이벤트를 감사 로그에 기록하고 통계를 수집한다.

## 4. 범위 / 비범위

- 포함: 새 IP/User-Agent 감지, 이메일 알림 발송, 알림 설정 (켜기/끄기)
- 비범위: 로그인 차단 (IP 차단은 별도 기능), ML 기반 이상 탐지

## 5. 사용자 시나리오

### 기본 흐름 — 새 IP 로그인 알림

1. 참여자: 사용자
2. 선행 조건: 이전 로그인 이력이 1건 이상 존재
3. 기본 흐름: 새 IP에서 로그인 성공 → 기존 IP 목록과 비교 → 불일치 시 이메일 알림 발송 ("새 위치에서 로그인되었습니다")
4. 예외 흐름: 이메일 발송 실패 → 재시도 큐, 알림 비활성화 사용자 → 스킵
5. 로그 포인트: `suspicious_login_detected`

## 6. UI / UX 방향

- 사용자 설정 → 보안 → "새 로그인 알림" 토글
- 이메일 내용: 시간, IP, 위치(GeoIP), User-Agent, "본인이 아닌 경우" 비밀번호 변경 링크

## 7. 데이터 / API / 권한 영향

- 테이블 확장: `sessions` 테이블의 ip_address, user_agent 활용
- 새 테이블: `known_login_locations` (user_id, ip_address, user_agent_hash, last_seen_at)
- 이메일 템플릿: `suspicious_login` 추가

## 8. 테스트 계획

- unit: IP 비교 로직, 알림 발송 조건 판단
- integration: 새 IP 로그인 → 알림 이메일 발송 확인
- e2e: 알림 비활성화 시 이메일 미발송 확인
