# 도메인 기반 자동 가입

- 문서명: 도메인 기반 자동 가입
- 목적: 허용된 이메일 도메인의 사용자가 초대 없이 조직에 자동으로 가입할 수 있게 한다.

## 1. 기능 개요

- 기능 이름: 도메인 기반 자동 가입
- 한 줄 정의: 조직이 허용 이메일 도메인을 등록하면 해당 도메인 사용자가 가입 시 자동으로 조직에 배정된다.

## 2. 문제 정의

- 같은 회사 이메일을 사용하는 직원이 일일이 초대를 받아야 하는 비효율이 있다.
- SSO 미도입 환경에서 도메인 기반 자동 가입이 온보딩을 간소화할 수 있다.

## 3. 목표

- 사용자 목표: 회사 이메일로 가입하면 자동으로 소속 조직에 배정된다.
- 운영 목표: 허용 도메인을 관리하고 무분별한 가입을 방지한다.

## 4. 범위 / 비범위

- 포함: 허용 도메인 CRUD, 가입 시 도메인 매칭 → 자동 조직 배정, 기본 역할 부여
- 비범위: 도메인 소유권 DNS 인증, 여러 조직에 동시 배정

## 5. 사용자 시나리오

### 기본 흐름 — 도메인 자동 가입

1. 참여자: 신규 사용자
2. 선행 조건: 조직에 허용 도메인이 등록됨
3. 기본 흐름: `POST /auth/signup` (email: user@company.com) → company.com 도메인 매칭 → 계정 생성 + 해당 조직 멤버십 자동 생성 (MEMBER 역할)
4. 예외 흐름: 허용 도메인 미등록 → 조직 미배정 (일반 가입), 도메인 중복 등록 → 첫 번째 조직에 배정
5. 로그 포인트: `domain_auto_join`

## 6. UI / UX 방향

- Admin Console → 조직 설정 → 허용 도메인 관리 (추가/삭제)
- 가입 완료 시 "company 조직에 자동으로 추가되었습니다" 안내

## 7. 데이터 / API / 권한 영향

- 새 테이블: `org_allowed_domains` (org_id, domain, created_at)
- 새 API: `GET/POST/DELETE /organizations/:orgId/allowed-domains`
- 가입 로직 확장: 도메인 매칭 → 멤버십 자동 생성

## 8. 테스트 계획

- unit: 도메인 추출, 매칭 로직, 중복 도메인 처리
- integration: 도메인 등록 → 해당 도메인으로 가입 → 멤버십 자동 생성 확인
- e2e: 허용 도메인 없는 이메일로 가입 → 조직 미배정 확인

## 9. 릴리즈 고려사항
