# 민감 정보 마스킹

- 문서명: 민감 정보 마스킹
- 목적: 로그에 민감한 정보가 노출되지 않도록 자동 마스킹 처리를 적용한다.
- 적용 범위: auth-service
- 상태: Draft
- Owner: Backend
- Reviewer: —
- 마지막 수정일: 2026-03-28
- 관련 SSOT 문서: gap-analysis.md

## 1. 기능 개요

- 기능 이름: 민감 정보 마스킹
- 한 줄 정의: 로그 serializer를 통해 비밀번호, 토큰, 시크릿 등 민감 필드를 자동으로 마스킹한다.

## 2. 문제 정의

- 디버깅 로그에 비밀번호, 토큰, API 키 등이 평문으로 노출될 위험이 있다.
- 컴플라이언스(GDPR, SOC 2)에서 로그 내 민감 정보 노출은 위반 사항이다.

## 3. 목표

- 사용자 목표: 해당 없음 (인프라 기능)
- 운영 목표: 모든 로그 출력에서 민감 필드가 자동 마스킹되어 안전한 로깅을 보장한다.

## 4. 범위 / 비범위

- 포함: 로그 serializer 구현, 민감 필드 목록 관리, stdout/파일 로그 모두 적용
- 비범위: DB 내 데이터 암호화, 네트워크 레벨 암호화

## 5. 사용자 시나리오

### 기본 흐름 — 자동 마스킹

1. 참여자: 시스템
2. 선행 조건: 로거 초기화 시 마스킹 serializer 등록
3. 기본 흐름: 로그 출력 시 → serializer가 민감 필드 탐지 → `***REDACTED***`로 치환
4. 예외 흐름: 알려지지 않은 필드명 → 패턴 매칭 (정규식)으로 추가 감지
5. 로그 포인트: 해당 없음 (로깅 인프라 자체)

## 6. UI / UX 방향

- 해당 없음 (백엔드 인프라 기능)

## 7. 데이터 / API / 권한 영향

- 코드 변경: pino/winston serializer 커스텀
- 마스킹 대상 필드: password, secret, token, access_token, refresh_token, authorization, cookie, mfa_token
- 설정: 환경변수로 추가 마스킹 필드 지정 가능 (`LOG_MASK_FIELDS`)

## 8. 테스트 계획

- unit: serializer에 민감 데이터 입력 → 마스킹 출력 확인
- unit: 중첩 객체 내 민감 필드 마스킹 확인
- integration: 실제 로그인 요청 로그에서 비밀번호 미노출 확인

## 9. 릴리즈 고려사항

- release note: 로그 민감 정보 자동 마스킹 적용
- ops: 기존 로그 포맷과의 호환성 확인
