## 용어 정리 (Project → Product)

auth-service에서 "Project"는 "Product"가 올바른 용어이다. Product는 auth-service가 관리하는 엔티티(MediLabel, INGRADIENT Platform 등)이고, Project는 ingradient-platform 내부 개념이다.

- [o] 파일명 변경: `features/project_membership.md` → `features/product_membership.md`
- [o] `features/README.md` — `project_membership.md` 링크 5곳을 `product_membership.md`로 변경

> `features/project_setup.md`는 "개발 프로젝트 초기화"를 의미하므로 Product와 무관. 파일명 유지.

## 새 기능

### Settings 페이지 (시스템 기본값 관리)

시스템 전체에 적용되는 기본 설정을 Admin이 관리하는 페이지. 조직에서 별도 설정하지 않으면 이 기본값이 적용된다.

- [o] 보안 정책 기본값 (비밀번호 정책, MFA, 계정 잠금, 세션 제한, 셀프 가입) → [system_settings.md](../features/system_settings.md)
- [o] 토큰/세션 설정 (Access Token TTL, Refresh Token TTL, Offline Entitlement 기본 유효 기간) → [system_settings.md](../features/system_settings.md)
- [o] 시스템 설정 (시스템 이름/로고, 기본 역할, 초대 유효 기간, 가입 코드 만료) → [system_settings.md](../features/system_settings.md)
- [o] 알림 설정 (이메일 발송 방식 SMTP/SendGrid/SES, 발신자, 웹훅 글로벌 설정) → [system_settings.md](../features/system_settings.md)
- [o] 라이선스 기본값 (기본 플랜, maxUsers, maxDevices, 오프라인 설정) → [system_settings.md](../features/system_settings.md)

참조: [wireframes/settings.md](../wireframes/settings.md)

### 사용자 승인 흐름 (Pending 상태)

사용자 생성 시 Pending 상태로 시작하고, Admin이 승인하면 Active로 전환되어 로그인 가능해지는 흐름.

- [o] 사용자 상태에 PENDING 추가 (기존: ACTIVE, DISABLED, INVITED, LOCKED) → [user_management.md](../features/user_management.md)
- [o] 승인 API (`POST /users/:id/approve` — Pending → Active 전환) → [user_management.md](../features/user_management.md)
- [o] 사용자 목록에서 Pending 행에 인라인 "승인" 버튼 → [user_management.md](../features/user_management.md)
- [o] 사용자 상세 모달에서 Pending일 때 "승인" 버튼 표시 → [user_management.md](../features/user_management.md)

참조: [wireframes/users.md](../wireframes/users.md), [user_scenarios.md](../user_scenarios.md) §사용자 온보딩

### 사용자 상세 모달 개선

## 버그 수정

### 로그인 MFA/비밀번호 만료 응답이 200으로 반환되던 문제

- MFA 활성 계정 또는 비밀번호 만료 계정 로그인 시 `{ mfa_required: true, ... }` 등을 HTTP 200으로 반환하여, 클라이언트가 빈 토큰 세션을 생성하는 문제
- 수정: HTTP 403 + `MFA_REQUIRED` / `PASSWORD_EXPIRED` 에러 코드로 변경
- 관련 파일: `auth.service.ts`, platform `auth.ts`
- 문서 반영: `api_contract.md` 에러 목록, `mfa_totp.md` 응답 형태

## 스키마 변경 — ProductMembership 개인 사용자 지원

개인 사용자(조직 미소속)도 Product에 직접 접근할 수 있도록 ProductMembership 스키마를 변경한다.

- [ ] ProductMembership에 `userId` 컬럼 추가 (FK → User, 필수)
- [ ] `membershipId`를 optional로 변경 (조직 소속 시 연결, 개인은 null)
- [ ] unique constraint 변경: `(membership_id, project_id)` → `(user_id, project_id)`
- [ ] 백엔드 service/repository 수정 — membership 없이도 Product 멤버 추가 가능
- [ ] 프론트엔드 — 조직 선택 없이도 Product 추가 UI 활성화
- [ ] GDPR 삭제 로직에 userId 직접 삭제 추가
