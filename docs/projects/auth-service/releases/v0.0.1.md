# v0.0.1

- 상태: Released
- Owner: Auth
- 마지막 수정일: 2026-03-29

## 버전 목표

ingradient-platform에서 인증/인가 책임을 분리하여 독립 서비스로 추출한다. 기본 인증, 사용자/조직 관리, 라이선스 시스템의 초기 기반을 구축한다.

## 프로젝트 초기화

- [o] Fastify 5 + TypeScript 프로젝트 스캐폴딩
- [o] Prisma ORM + PostgreSQL 16 연동
- [o] Docker Compose 환경 (PostgreSQL + Redis)
- [o] ESM 모듈 시스템 설정
- [o] 환경변수 관리 (dotenv + Zod 스키마 검증)
- [o] 구조화 로그 (Fastify pino, JSON)
- [o] 에러 핸들링 (HttpError 클래스, 일관된 응답 구조)
- [o] 입력 검증 프레임워크 (Zod + validateBody 유틸)

## 인증 기반

- [o] JWT 발급 (HS256, jose 라이브러리)
- [o] Access Token (기본 1시간 TTL)
- [o] Refresh Token (48바이트 랜덤, SHA256 해시 저장, 기본 30일)
- [o] 로그인 (`POST /auth/login` — login_id + password)
- [o] 로그아웃 (`POST /auth/logout`)
- [o] 토큰 검증 (`POST /auth/verify` — downstream 서비스용)
- [o] Guard 미들웨어 (requireBearer, requireAdmin, adminOrBootstrap, bearerOrBootstrap)
- [o] Bootstrap Secret 인증 (`X-Bootstrap-Secret` 헤더, 서비스 간 통신)

## 사용자 / 조직 / Product

- [o] 사용자 CRUD (생성, 조회, 수정, soft delete)
- [o] 조직 CRUD (code 기반 식별, ACTIVE/SUSPENDED/DISABLED)
- [o] Product CRUD (조직 하위, code 기반 식별)
- [o] 멤버십 (User → Organization 매핑 + Role)
- [o] Product 멤버십 (Membership 하위, Product별 역할)

## 역할 / 권한

- [o] 역할 CRUD (GLOBAL / ORGANIZATION / PRODUCT scope)
- [o] 시스템 역할 시드 (ADMIN, ORGANIZER, MEMBER)
- [o] 권한 정의 (resource:action 패턴)
- [o] 역할-권한 매핑 (다대다)
