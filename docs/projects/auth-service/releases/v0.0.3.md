## 버전 목표

gap-analysis에서 도출된 나머지 미구현 항목을 우선순위별로 구현한다. 엔터프라이즈 보안 강화, 셀프서비스 온보딩, 운영 가시성, 규정 준수를 목표로 한다.

## P1 — 엔터프라이즈 보안

- [o] MFA/2FA (TOTP) → [mfa_totp.md](../features/mfa_totp.md)
- [o] 비밀번호 정책 강화 (복잡도, 이력, 만료, 블랙리스트, 조직별) → [password_policy_enhancement.md](../features/password_policy_enhancement.md)
- [o] 세션 생명주기 (만료 정리, 동시 제한, 토큰 블랙리스트) → [session_lifecycle.md](../features/session_lifecycle.md)
- [o] Admin 대시보드 → [admin_dashboard.md](../features/admin_dashboard.md)
- [o] JWT 키 rotation (Access Token) → [jwt_key_rotation.md](../features/jwt_key_rotation.md)
- [o] 조직별 보안 설정 (MFA 필수, 세션 타임아웃 per org) → [org_security_settings.md](../features/org_security_settings.md)

## 배포 계획

- P1 → P2 → P3 순서로 단계별 구현 및 배포
- 각 기능은 독립적으로 배포 가능하도록 feature flag 또는 환경변수로 제어
- migration: 기능별 Prisma 마이그레이션 개별 관리
- 주요 신규 의존성: otplib (TOTP), prom-client (메트릭), saml2-js (SAML)
