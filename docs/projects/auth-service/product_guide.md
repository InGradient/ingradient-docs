## 이 문서의 역할

이 문서는 **개발 전**에 작성한다. 기술적 상세(API, DB, 아키텍처)가 아닌 **사용자가 할 수 있는 것**을 화면 단위로 정리한다. features/ 폴더의 상세 기획보다 가볍게, 일반인도 이해할 수 있는 수준으로 작성한다.

`user_guide.md`와의 차이: product_guide는 "뭘 할 수 있는가" (개발 전 기획 입력), user_guide는 "어떻게 쓰면 되는가" (개발 후 사용법).

## 제품 운영 방식

이 서비스가 어떤 방식으로 동작하는지 먼저 이해한다. 각 화면 설명의 맥락이 된다.

- **회원가입은 셀프 가입이 아닌 Admin 초대 또는 가입 코드로만 가능하다.** 사용자 생성/초대 시 Pending 상태로 시작하며, Admin이 승인하면 Active가 되어 로그인 가능해진다
- **비밀번호 분실 시 두 가지 방법이 있다.** 이메일로 직접 재설정하거나, Admin에게 초기화를 요청하여 임시 비밀번호를 받을 수 있다
- **소속 없이 개인 사용자로도 가입할 수 있다.** 개인 사용자도 Product에 직접 접근할 수 있다. 조직에 속하면 조직 단위로 관리되고, 소속 없는 개인 사용자는 나중에 조직에 초대되거나 가입 코드로 합류할 수도 있다
- **역할은 조직 레벨과 Product 레벨로 나뉜다.** 조직에서의 역할과 Product에서의 역할이 별도로 부여된다. 개인 사용자는 조직 역할 없이 Product 역할만 가질 수 있다
- **비밀번호 5회 실패 시 15분 자동 잠금된다.** 시간이 지나면 자동 해제되거나 Admin이 수동 해제할 수 있다
- **오프라인 디바이스는 별도 라이선스를 발급해야 사용 가능하다.** 인터넷 없는 환경의 Edge 디바이스용
- **온라인 디바이스는 시스템에 연결하면 자동 등록된다.** 별도 승인 절차 없이 바로 사용 가능
- **조직별로 보안 정책을 다르게 설정할 수 있다.** MFA 필수, 비밀번호 규칙, 세션 제한 등을 조직마다 다르게 운영
- **SSO를 설정하면 외부 계정(Google, Microsoft 등)으로 로그인 가능하다.** 첫 로그인 시 계정이 자동 생성될 수 있다 (JIT)
- **모든 주요 행동은 감사 로그에 기록된다.** 로그인, 멤버 변경, 디바이스 등록/폐기, 초대 등

## Admin 대시보드

System Admin 전용. 전체 시스템의 사용자, 조직, Product, 디바이스, 라이선스를 관리한다.

### 로그인

- Login ID + 비밀번호로 관리자 로그인
- INGRADIENT 로고가 표시됨

### 사용자 관리

- 전체 사용자 목록 조회 (Login ID, 이름, 이메일, 역할, 상태, 마지막 로그인)
- 사용자 검색 (Login ID, 이름)
- 역할별 필터 (Admin, Member)
- 새 사용자 생성 (Login ID, 비밀번호, 이름, 이메일. 소속 조직/Product는 선택 — 조직 없이 개인 사용자로도 생성 가능. 조직 선택 시 다중 Product 배정 및 Product별 역할 지정)
- 사용자 정보 수정 (이름, 이메일, 역할 변경)
- 비밀번호 초기화 (임시 비밀번호 생성)
- 승인 대기 사용자 승인 (Pending → Active)
- 사용자 활성화/비활성화
- 강제 로그아웃 (모든 세션 종료)
- 사용자 삭제

### 조직 관리

- 전체 조직 목록 조회 (코드, 이름, 상태)
- 조직 요약 (총, 활성, 비활성 수)
- 조직 검색 (이름, 코드)
- 새 조직 생성 (고유 코드, 이름)
- 조직 정보 수정 (이름, 상태 변경)
- 조직 멤버 관리:
  - 멤버 목록 조회
  - 멤버 추가 (사용자 선택, 역할 부여)
  - 멤버 역할 변경
  - 멤버 제거

### Product 관리

- 전체 Product 목록 조회 (코드, 이름, 상태)
- Product 검색 (이름, 코드)
- 새 Product 생성 (코드, 이름)
- Product 정보 수정 (이름, 상태)
- Product 멤버 조회
- Product 삭제

### 라이선스 & 디바이스

- 온라인 디바이스와 오프라인 라이선스를 함께 조회
- 요약 (총, 활성, 온라인, 오프라인 수)
- 검색 (조직, 디바이스 이름, UID, 상태)
- 모드별 필터 (Online / Offline)
- 온라인 디바이스:
  - 디바이스가 시스템에 연결하면 자동 등록됨
  - 디바이스 폐기
  - 폐기된 디바이스 삭제
- 오프라인 라이선스:
  - 새 라이선스 발급 (하드웨어 fingerprint 입력, 라벨, 조직, 만료일 설정)
  - 라이선스 키 복사
  - 라이선스 폐기

### 세션 관리

- 전체 시스템의 활성 세션 목록 조회
- 세션 정보 (사용자, Login ID, IP, 브라우저, 생성/마지막 사용 시각)
- 개별 세션 강제 종료

### 감사 로그

- 상단에 시각화 그래프:
  - 시간대별 이벤트 추이 (로그인 성공/실패, 멤버 변경 등 — 이상 징후 발견용)
  - 이벤트 유형별 분포
- 하단에 상세 로그 목록:
  - 시스템에서 발생한 모든 이벤트 조회
  - 이벤트 유형별 필터 (로그인 성공/실패, 로그아웃, 사용자 생성/수정/삭제, 멤버 추가/제거, 디바이스 등록/폐기, 초대 생성/수락, 엔타이틀먼트 발급/취소 등)
  - 날짜 범위 필터
  - 감사 로그 내보내기 (CSV, JSON)

### 초대 관리

- 조직 선택 후 해당 조직의 초대 목록 조회
- 새 초대 생성 (이메일, 역할, 유효 기간 설정)
- 대기 중인 초대 취소

### Settings

시스템 전체에 적용되는 기본 설정. 조직에서 별도 설정하지 않으면 이 기본값이 적용된다.

- 보안 정책 기본값 (비밀번호 정책, MFA, 계정 잠금, 세션 제한, 셀프 가입)
- 토큰/세션 설정 (Access Token TTL, Refresh Token TTL, Offline Entitlement 기본 유효 기간)
- 시스템 설정 (시스템 이름/로고, 기본 역할, 초대 기본 유효 기간, 가입 코드 기본 만료)
- 알림 설정 (이메일 발송 방식 — SMTP/SendGrid/SES, 웹훅 글로벌 설정)
- 라이선스 기본값 (기본 플랜, maxUsers, maxDevices, 오프라인 설정)

## 일반 사용자 기능

모든 사용자가 사용할 수 있는 기능. 별도 Admin 대시보드 없이 API 또는 연동 제품(platform, edge 등)을 통해 사용한다.

### 인증

- Login ID + 비밀번호로 로그인
- 비밀번호 5회 연속 실패 시 15분 자동 잠금
- 자동 토큰 갱신 (사용자가 의식하지 않아도 세션 유지)
- 로그아웃
- 전체 디바이스 로그아웃 (모든 세션 한번에 종료)

### 비밀번호 관리

- 비밀번호 변경 (현재 비밀번호 확인 필수)
- 비밀번호 찾기 (이메일로 재설정 링크 발송)
- 비밀번호 재설정 (링크를 통해 새 비밀번호 설정)
- 또는 Admin에게 초기화 요청 → Admin이 임시 비밀번호를 생성하여 전달 → 로그인 후 본인이 변경
- 비밀번호 규칙: 8자 이상, 문자 1개 이상, 숫자 1개 이상

### 2단계 인증 (MFA)

- TOTP 앱(Google Authenticator 등)으로 2단계 인증 설정
- 로그인 시 TOTP 코드 입력
- 백업 코드 발급 (앱 분실 시 사용)
- 2단계 인증 해제

### 세션 관리

- 내 활성 세션 목록 조회 (어떤 디바이스에서 로그인 중인지)
- 개별 세션 종료 (특정 디바이스에서 로그아웃)

### SSO (Single Sign-On)

- Google, Microsoft 등 외부 계정으로 로그인
- 조직 관리자가 SSO를 설정하면 해당 조직 멤버는 SSO로 로그인 가능
- 첫 SSO 로그인 시 계정이 자동 생성될 수 있음 (JIT Provisioning)

## 조직 관리자 기능

자기 조직 내에서 사용할 수 있는 관리 기능.

### 멤버 관리

- 조직 멤버 목록 조회
- 멤버 추가 (역할 지정)
- 멤버 역할 변경
- 멤버 제거

### 초대

- 이메일로 조직 초대 발송 (역할 지정, 유효 기간 설정)
- 초대 재발송
- 초대 취소

### 가입 코드

- 공개 가입 코드 생성 (코드를 아는 사람은 누구나 가입)
- 최대 사용 횟수, 만료일 설정 가능
- 가입 코드 비활성화

### 디바이스 관리

- 조직 내 디바이스 목록 조회
- 디바이스 폐기

### 오프라인 접근

- Edge 디바이스에 오프라인 접근 토큰 발급
- 오프라인 토큰 취소
- 오프라인 토큰 갱신
- 조직 내 오프라인 토큰 목록 조회

### 보안 설정

- 조직 보안 정책 조회/수정:
  - MFA 필수 여부
  - 비밀번호 요구사항 (최소 길이, 대문자, 소문자, 숫자, 특수문자)
  - 비밀번호 만료 (최대 기간, 이력 수)
  - 세션 제한 (최대 동시 세션, 타임아웃)
  - 셀프 가입 허용 여부

### SSO 설정

- SSO IdP 설정 (Google, Microsoft 등)
- JIT 자동 계정 생성 활성화/비활성화
- JIT 시 기본 역할 지정
- IdP 설정 삭제
