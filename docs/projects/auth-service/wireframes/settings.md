# Settings

## 진입 경로

- 사이드바 "Settings" 클릭 → `/admin/settings`

## 화면 구조

탭 또는 섹션 기반으로 설정 항목을 그룹핑한다. 각 섹션은 한 줄씩 설정 항목 + 입력/토글로 구성한다.

### 탭 1 — 보안 정책 (시스템 기본값)

조직에서 별도 설정하지 않으면 이 기본값이 적용된다.

**인증:**
- MFA 필수 여부 (토글, 기본: OFF)
- 셀프 가입 허용 여부 (토글, 기본: OFF)
- 계정 잠금 — 로그인 실패 횟수 (숫자 입력, 기본: 5)
- 계정 잠금 — 잠금 시간 (분 단위, 기본: 15)

**비밀번호 정책:**
- 최소 길이 (숫자 입력, 기본: 8)
- 대문자 필수 (토글, 기본: OFF)
- 소문자 필수 (토글, 기본: OFF)
- 숫자 필수 (토글, 기본: ON)
- 특수문자 필수 (토글, 기본: OFF)
- 비밀번호 만료 기간 (일 단위, 0이면 만료 없음, 기본: 0)
- 비밀번호 이력 수 (이전 N개 재사용 금지, 기본: 0)

**세션 제한:**
- 최대 동시 세션 수 (숫자 입력, 0이면 무제한, 기본: 0)
- 세션 타임아웃 (분 단위, 기본: 1440 = 24시간)

하단: "Save" 버튼 (변경사항 있을 때만 활성화)

### 탭 2 — 토큰 / 세션

**Access Token:**
- TTL (초 단위, 기본: 3600 = 1시간)

**Refresh Token:**
- TTL (일 단위, 기본: 30)

**Offline Entitlement:**
- 기본 유효 기간 (일 단위, 기본: 7)

하단: "Save" 버튼

### 탭 3 — 시스템

**브랜딩:**
- 시스템 이름 (텍스트 입력, 기본: "INGRADIENT")
- 로고 이미지 업로드 (Admin 대시보드 로고)

**기본값:**
- 새 사용자 생성 시 기본 역할 (드롭다운: Admin / Organizer / Member, 기본: Member)
- 초대 기본 유효 기간 (시간 단위, 기본: 72)
- 가입 코드 기본 만료 기간 (일 단위, 0이면 만료 없음, 기본: 0)

하단: "Save" 버튼

### 탭 4 — 알림

**이메일:**
- 이메일 발송 방식 (드롭다운: 비활성 / SMTP / SendGrid / SES)
- SMTP 선택 시: 호스트, 포트, 사용자, 비밀번호, TLS 여부
- SendGrid 선택 시: API Key
- SES 선택 시: Region, Access Key, Secret Key
- 발신자 이메일 (텍스트 입력)
- 발신자 이름 (텍스트 입력)
- "테스트 이메일 발송" 버튼

**웹훅:**
- 웹훅 글로벌 활성화 (토글)
- 기본 재시도 횟수 (숫자 입력, 기본: 3)
- 재시도 간격 (초 단위, 기본: 30)

하단: "Save" 버튼

### 탭 5 — 라이선스 기본값

새 조직에 라이선스를 할당할 때 사용되는 기본값.

- 기본 플랜 코드 (텍스트 입력, 기본: "standard")
- 기본 최대 사용자 수 (숫자 입력, 기본: 10)
- 기본 최대 디바이스 수 (숫자 입력, 기본: 5)
- 오프라인 기능 허용 (토글, 기본: ON)
- 오프라인 최대 유효 기간 (일 단위, 기본: 7)
- 오프라인 유예 모드 (드롭다운: read_only / disabled, 기본: read_only)

하단: "Save" 버튼

## 주요 인터랙션

- **탭 전환**: 저장하지 않은 변경사항이 있으면 "저장하지 않은 변경사항이 있습니다. 이동하시겠습니까?" 확인
- **"Save" 클릭**: 해당 탭의 변경사항만 저장 + 성공 토스트
- **"테스트 이메일 발송" 클릭**: 현재 설정으로 테스트 이메일 발송 → 성공/실패 토스트
- **로고 업로드**: 파일 선택 → 미리보기 표시 → Save 시 반영
- **숫자 입력 필드**: 음수 입력 불가, 최소/최대 범위 표시

## 상태별 화면

- loading: 각 탭 내용 스켈레톤
- 저장 성공: 성공 토스트 + Save 버튼 비활성화
- 저장 실패: 에러 토스트 + 에러 필드 하이라이트
- 이메일 테스트 성공: "테스트 이메일이 발송되었습니다" 토스트
- 이메일 테스트 실패: "이메일 발송에 실패했습니다: {에러}" 토스트
