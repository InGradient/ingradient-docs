# 세션 관리

## 진입 경로

- 사이드바 "Sessions" 클릭 → `/admin/sessions`

## 화면 구조

- 상단:
  - 페이지 제목 "Sessions"
  - 요약 카드: 총 활성 세션 수, 고유 사용자 수
  - 검색바 (사용자 이름, Login ID, IP)
  - "전체 세션 종료" 버튼 (위험 — 모든 사용자의 세션을 한번에 종료)
- 메인:
  - 테이블:
    - 사용자 이름
    - Login ID
    - IP 주소
    - User Agent (브라우저/OS 아이콘 + 텍스트)
    - 생성 시각
    - 마지막 사용 시각
    - 행마다 "종료" 버튼
  - 같은 사용자의 세션이 여러 개면 사용자 이름 옆에 세션 수 뱃지 표시 (예: "홍길동 (3)")
- 하단:
  - 페이지네이션 (20개씩)

## 주요 인터랙션

- **검색**: 사용자 이름, Login ID, IP로 실시간 필터링
- **개별 "종료" 클릭**: 확인 다이얼로그 ("이 세션을 종료하시겠습니까? 해당 디바이스에서 즉시 로그아웃됩니다") → 세션 종료 + 목록 갱신 + 토스트
- **"전체 세션 종료" 클릭**: 위험 확인 다이얼로그 (빨간색, "모든 사용자의 모든 세션이 종료됩니다. 전체 사용자가 재로그인해야 합니다") → 전체 종료 + 목록 갱신 + 토스트
- **행 클릭**: 세션 상세 정보를 인라인 확장 행으로 표시:
  - 세션 ID
  - Token Family ID
  - Client ID
  - 전체 User Agent 문자열
  - 생성 시각 (상세)
  - 마지막 사용 시각 (상세)

## 상태별 화면

- empty: "활성 세션이 없습니다" 안내
- loading: 요약 카드 스켈레톤 + 테이블 스켈레톤
- 검색 결과 없음: "조건에 맞는 세션이 없습니다" 안내
- 세션 종료 직후: 해당 행이 테이블에서 사라지며 요약 카드 갱신
