# Wireframes

- 문서명: Wireframes
- 목적: 개발 전에 각 화면의 구조, 요소 배치, 인터랙션을 설계한다. 디자이너, 기획자, 개발자가 화면 동작을 합의하는 용도.
- 상태: Draft
- Owner: Product / Design
- 마지막 수정일:

## 이 문서의 역할

| 문서 | 역할 |
|------|------|
| product_guide.md | "이 화면에서 뭘 할 수 있는가" (개요) |
| user_scenarios.md | "이 목적을 위해 어떤 화면을 거치는가" (흐름) |
| **wireframes/** | **"이 화면이 어떻게 생겨야 하는가"** (설계) |
| features/{기능}.md | 상세 기획 (시나리오, 예외 흐름, 테스트 계획) |
| user_guide.md | "이 화면을 어떻게 쓰면 되는가" (사용법, 개발 후) |

- 개발을 모르는 사람이나 디자이너가 읽고 화면을 이해할 수 있어야 한다
- 기술 용어는 최소화하고, 화면 구조와 사용자 행동 중심으로 작성한다
- 개발 전에 작성하여 합의한 후 UIUX 프로토타입이나 구현에 들어간다

## 작성 형식

화면별로 아래를 정리한다.

```markdown
### {화면 이름}

**진입 경로**: 어디에서 이 화면으로 들어오는가

**화면 구조**:
- 상단: (헤더, 제목, 액션 버튼 등)
- 메인: (테이블, 카드, 폼 등)
- 하단: (페이지네이션, 상태바 등)

**주요 인터랙션**:
- {버튼/액션}: {어떤 일이 일어나는가} (모달, 토스트, 페이지 이동 등)

**상태별 화면**:
- empty: (데이터 없을 때)
- loading: (로딩 중)
- error: (에러 발생 시)
```
