# ingradient-platform User Scenarios

## 시나리오 1: 새 프로젝트 시작

**목적**: 처음 사용하는 팀이 프로젝트를 만들고 데이터를 업로드할 수 있다.

1. 로그인 (`/login`) — auth-service 계정으로 인증
2. 프로젝트 생성 (`/create-project`) — 이름, 설명 입력
3. 클래스 정의 (`/classes`) — 분류 체계 생성 (예: 정상/불량/스크래치)
4. 데이터 업로드 (`/catalog`) — 이미지 파일 드래그앤드롭
5. 데이터셋 생성 (`/catalog`) — 업로드된 이미지를 데이터셋으로 묶기

## 시나리오 2: 팀원 온보딩

**목적**: 프로젝트에 새 팀원을 추가하고 적절한 권한을 부여한다.

1. 설정 (`/settings`) — 멤버 초대 (이메일 또는 Login ID)
2. 초대받은 사용자 — auth-service에서 회원가입/로그인
3. 프로젝트 접근 — 멤버십 확인 후 대시보드 진입
4. 역할 조정 — Owner가 Viewer → Editor로 변경 (필요 시)

## 시나리오 3: 라벨링 작업

**목적**: 업로드된 이미지에 클래스 라벨을 붙인다.

1. 카탈로그 (`/catalog`) — 라벨링할 데이터셋 선택
2. 갤러리 (`/gallery`) — 이미지 클릭하여 상세 모달 열기
3. 라벨 편집 — 클래스 선택, 바운딩 박스 또는 분류 라벨 지정
4. 저장 — 라벨 데이터 저장
5. 코멘트 — 애매한 이미지에 대해 팀원에게 멘션으로 의견 요청

## 시나리오 4: AI 모델 학습

**목적**: 라벨링된 데이터로 객체 검출 모델을 학습시킨다.

1. 학습 페이지 (`/training`) — 학습 요청 생성
2. 데이터셋 선택 — 학습에 사용할 데이터셋과 클래스 매핑
3. 파라미터 설정 — 모델 타입 (YOLO), 에폭, 배치 사이즈 등
4. 학습 시작 — ingradient-ai로 Job 전달
5. 진행률 모니터링 — 큐 상태, 손실 그래프 확인
6. 완료 — 모델 라이브러리에 자동 등록

## 시나리오 5: 모델 분석 및 배포

**목적**: 학습된 모델의 성능을 확인하고 Edge 디바이스에 배포한다.

1. 분석 (`/analysis`) — 학습된 모델의 추론 결과 확인
2. 이미지별 예측 비교 — 정답 vs 예측 시각적 비교
3. 모델 배정 (`/models`) — 프로젝트에 최적 모델 배정
4. Edge 패키지 생성 (`/edge`) — .ige 패키지 빌드
5. 디바이스 배포 — 패키지 다운로드 → Edge 디바이스에 설치

## 시나리오 6: 데이터 내보내기

**목적**: 외부 도구에서 사용할 수 있는 형태로 데이터를 내보낸다.

1. 카탈로그 (`/catalog`) — 내보낼 데이터셋 선택
2. Export 요청 — COCO JSON 형식 선택
3. ZIP 다운로드 — 이미지 + 어노테이션 JSON
4. 외부 도구에서 사용 — YOLO, Detectron2 등에서 학습

## 시나리오 7: 대시보드 커스터마이징

**목적**: 프로젝트 현황을 한눈에 파악할 수 있는 대시보드를 설정한다.

1. 대시보드 (`/`) — 위젯 추가/제거
2. 드래그앤드롭 — 위젯 위치 조정
3. 설정 저장 — 사용자별 레이아웃 저장
