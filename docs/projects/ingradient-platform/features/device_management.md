# 디바이스 관리 / 오프라인 토큰 (Device Management & Offline Tokens)

## 기능 개요

Edge 디바이스 등록, 라이선스 관리, 오프라인 토큰 발급을 하는 기능이다. 조직 관리자가 Admin 페이지에서 관리한다.

## 문제 정의

오프라인 공장 환경에서 Edge 디바이스가 인터넷 없이 인증하려면 사전 발급된 오프라인 토큰이 필요하다. 디바이스 등록/폐기/라이선스 관리를 중앙에서 통제해야 한다.

## 범위 / 비범위

- **포함**: 디바이스 등록/조회/폐기, 오프라인 토큰 생성, 라이선스 정보
- **제외**: 디바이스 실행 제어 → ingradient-edge, 라이선스 발급 → auth-service

## 사용자 시나리오

### 디바이스 등록 확인

- **기본 흐름**:
  1. Admin 페이지(`/admin`) → Devices 탭
  2. 등록된 디바이스 목록: UID, 이름, 등록일, 상태
  3. 디바이스 상세 확인
- **디바이스 폐기**: 더 이상 사용하지 않는 디바이스 비활성화

### 오프라인 토큰 발급

- 오프라인 환경용 인증 토큰 생성
- 유효 기간 설정 가능
- Edge 패키지에 포함되어 현장에서 오프라인 인증에 사용

## UI / UX 방향

- Admin 페이지의 Devices 탭
- 디바이스 테이블: UID, 이름, 등록일, 마지막 접속, 상태
- 폐기/삭제 버튼 (확인 대화상자)
- 라이선스 정보: 플랜, 만료일, 최대 디바이스 수

## 데이터 / API / 권한 영향

- auth-service의 디바이스/라이선스 API 연동
- 조직 관리자(org admin) 역할에게만 노출
- Edge 패키지 export 시 오프라인 토큰 포함 여부 결정

## 테스트 계획

- 디바이스 목록 조회 → 등록된 디바이스 표시
- 디바이스 폐기 → 상태 변경 확인
- 오프라인 토큰 생성 → Edge 패키지에 포함
