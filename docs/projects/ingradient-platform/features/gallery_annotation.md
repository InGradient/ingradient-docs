# 갤러리 / 어노테이션 (Gallery & Annotation)

## 기능 개요

이미지를 그리드로 탐색하고, 상세 모달에서 바운딩 박스, 포인트, 분류 라벨을 편집하는 핵심 작업 화면이다.

## 문제 정의

AI 학습 데이터의 품질은 라벨링 정확도에 직결된다. 대량의 이미지를 빠르게 탐색하고, 직관적으로 어노테이션할 수 있는 UI가 필요하다.

## 범위 / 비범위

- **포함**: 이미지 그리드 뷰, 상세 모달, 바운딩 박스/포인트/분류 라벨 편집, AI 자동 감지, 코멘트
- **제외**: 데이터셋 구조 관리 → Catalog, 학습 실행 → Training

## 사용자 시나리오

### 이미지 탐색

- **기본 흐름**:
  1. 갤러리 페이지 진입 → 가상화된 이미지 그리드 표시
  2. 스크롤로 수십만 장 탐색 (무한 스크롤)
  3. 필터: 라벨 상태, 클래스, 데이터셋
  4. 정렬: 업로드일, 파일명, 크기

### 어노테이션 작업

- **기본 흐름**:
  1. 이미지 클릭 → 상세 모달 열림
  2. 어노테이션 타입 선택:
     - **바운딩 박스**: 드래그로 영역 지정, 클래스 할당
     - **포인트**: 클릭으로 좌표 지정, 클래스 할당
     - **분류**: 이미지 전체에 클래스 라벨 지정
  3. 저장 → 라벨 데이터 업데이트
  4. 좌/우 화살표로 다음 이미지 이동
- **예외 흐름**:
  - 저장 실패 → 에러 표시, 데이터 유지
  - 동시 편집 → 최신 저장이 우선

### AI 자동 감지

- **기본 흐름**:
  1. 상세 모달에서 "자동 감지" 실행
  2. 프로젝트에 배정된 YOLO 모델로 객체 감지
  3. 예측 결과(바운딩 박스 + 신뢰도) 오버레이 표시
  4. 사용자가 확인/수정 후 저장

## UI / UX 방향

### 그리드 뷰

- 가상화된 무한 스크롤 그리드 (성능 최적화)
- 썸네일에 라벨 상태 표시 (라벨 있음/없음)
- 다중 선택 (Ctrl+클릭, Shift+범위)
- 선택 바: 선택 개수, 일괄 삭제/이동/클래스 할당

### 상세 모달

| 영역 | 구성 |
|------|------|
| 좌측 | 원본 이미지 + 바운딩 박스/포인트 오버레이 |
| 우측 상단 | 메타데이터 (해상도, 크기, 업로더, 날짜, 소스) |
| 우측 중앙 | 클래스 선택 패널 (색상 코딩) |
| 우측 하단 | 코멘트 스레드 (@멘션 지원) |

- 신뢰도 점수 표시 (AI 예측 결과)
- 히트맵 시각화 (모델 출력)
- 시퀀스 그룹 이미지 연속 탐색

## 데이터 / API / 권한 영향

| API | 용도 |
|-----|------|
| `GET /api/images` | 이미지 목록 (cursor pagination) |
| `GET /api/images/{id}` | 이미지 상세 |
| `PATCH /api/images/{id}` | 어노테이션 업데이트 |
| `POST /api/images/{id}/classification` | 분류 라벨 추가 |
| `POST /api/projects/{id}/detect` | AI 객체 감지 실행 |

- 바운딩 박스 좌표: 정규화 (0~1), `{ x, y, w, h, classId }`
- 포인트 좌표: 정규화 (0~1), `{ x, y, classId }`
- Consensus 라벨: 다수 라벨러의 결과를 병합한 최종 라벨

## 테스트 계획

- 수만 장 이미지 그리드 스크롤 성능
- 바운딩 박스 그리기 → 저장 → 재로딩 시 위치 일치
- AI 자동 감지 결과 오버레이 정상 표시
- 분류 라벨 할당/제거 → 필터에 반영
- 코멘트 @멘션 → 대상 사용자에게 표시
