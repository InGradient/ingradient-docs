# 카탈로그 / 데이터셋 (Catalog & Dataset)

## 기능 개요

프로젝트 내 이미지 데이터를 데이터셋 단위로 조직하고, 업로드, 검색, 필터링, 일괄 작업을 수행하는 중심 화면이다.

## 문제 정의

수만~수십만 장의 이미지를 효율적으로 관리하려면 논리적 그룹핑(데이터셋), 빠른 검색, 일괄 작업이 필수다. 학습용 데이터셋 구성(train/eval/test split)도 이 화면에서 이루어져야 한다.

## 범위 / 비범위

- **포함**: 데이터셋 CRUD, 이미지 업로드/삭제/이동/복사, 검색/필터/정렬, split 관리, 썸네일, 클래스 필터
- **제외**: 이미지 상세 편집(라벨링) → Gallery, 내보내기 실행 → Export

## 사용자 시나리오

### 데이터셋 생성 및 이미지 업로드

- **기본 흐름**:
  1. 좌측 패널에서 "데이터셋 추가" 클릭
  2. 이름 입력 후 생성
  3. 중앙 영역에 이미지 드래그앤드롭 (다중 파일)
  4. 업로드 진행률 표시, 썸네일 자동 생성
  5. 중복 파일은 file_hash로 감지하여 건너뜀
- **예외 흐름**:
  - 지원하지 않는 형식 → 에러 표시, 나머지 계속 진행
  - 네트워크 오류 → 실패 파일 목록 표시, 재시도 가능

### 이미지 일괄 작업

- **선택**: 클릭, Ctrl+클릭 (개별 추가), Shift+클릭 (범위 선택)
- **작업**: 다른 데이터셋으로 이동/복사, split 변경 (train/eval/test), 일괄 삭제
- **삭제 권한**: `data.delete` 권한 있으면 모든 이미지, 없으면 본인 업로드만

### 검색과 필터

- 정렬: 업로드일 (오래된/최신), 파일명, 파일 크기, split 순서
- 필터: 라벨 상태 (전체/라벨됨/미라벨), 클래스별
- 데이터셋별 요약 통계: train/eval/test 개수, 라벨 완료 수

## UI / UX 방향

| 영역 | 구성 |
|------|------|
| 좌측 패널 | 데이터셋 목록, 요약 통계 (train/eval/test), 생성/복제/삭제 |
| 중앙 영역 | 가상화된 이미지 그리드 (수십만 장 대응), 정렬/필터 툴바, 드래그앤드롭 업로드 |
| 우측 패널 | 클래스 풀 (색상 스와치), 멤버 목록, 클래스 검색 |

- **상태**: empty (데이터셋 없음), loading (이미지 로딩), 업로드 진행률 오버레이
- 이미지 썸네일에 해상도/크기 오버레이 표시

## 데이터 / API / 권한 영향

| API | 용도 |
|-----|------|
| `GET /api/datasets` | 데이터셋 목록 |
| `POST /api/datasets` | 생성 |
| `PATCH /api/datasets/{id}` | 수정 (이름 변경 등) |
| `DELETE /api/datasets/{id}` | 삭제 |
| `GET /api/images?cursor=&limit=200&sort=` | 이미지 목록 (cursor pagination) |
| `POST /api/upload` | 이미지 업로드 (multipart) |
| `POST /api/images/delete` | 일괄 삭제 |

- 이미지와 데이터셋은 **N:N 관계** — 하나의 이미지가 여러 데이터셋에 속할 수 있음
- 데이터셋별 task_type: `classification`, `object_detection`, `segmentation`, `point`

## 테스트 계획

- 빈 프로젝트에서 데이터셋 생성 → 이미지 업로드 → 목록 표시
- 중복 파일 업로드 시 건너뛰기 동작
- 수천 장 이미지 스크롤 성능 (가상화 그리드)
- 데이터셋 간 이미지 이동 → 원본 데이터셋에서 제거 확인
- 권한 없는 사용자의 삭제 시도 → 본인 업로드만 삭제
