# 코멘트 / 협업 (Comments & Collaboration)

## 기능 개요

이미지 단위로 코멘트를 남기고 @멘션으로 팀원에게 의견을 요청하는 협업 기능이다.

## 문제 정의

라벨링 작업 중 애매한 이미지에 대해 팀원과 소통할 수 있어야 한다. 별도 메신저 없이 이미지 컨텍스트 안에서 바로 논의해야 효율적이다.

## 범위 / 비범위

- **포함**: 이미지 코멘트 CRUD, @멘션, 스레드 형태 대화
- **제외**: 실시간 알림 → 향후, 코멘트 기반 태스크 할당 → 향후

## 사용자 시나리오

### 코멘트 작성

- **기본 흐름**:
  1. Gallery에서 이미지 클릭 → 상세 모달 열림
  2. 우측 하단 코멘트 영역에서 텍스트 입력
  3. `@이름`으로 팀원 멘션 (자동완성)
  4. 전송 → 코멘트 스레드에 표시
- **예외 흐름**:
  - 빈 코멘트 전송 시도 → 차단
  - 네트워크 오류 → 재시도 안내

### 코멘트 관리

- 본인 코멘트 편집/삭제 가능
- 작성자, 작성 시간 표시
- 시간순 정렬

## UI / UX 방향

- Gallery 상세 모달의 우측 하단에 위치
- 스레드 형태 (시간순)
- @멘션 시 자동완성 팝업 (프로젝트 멤버 목록)
- 편집 시 인라인 편집 모드
- 삭제 시 확인 대화상자

## 데이터 / API / 권한 영향

| API | 용도 |
|-----|------|
| `GET /api/images/{id}/comments` | 코멘트 목록 |
| `POST /api/images/{id}/comments` | 코멘트 작성 |
| `PATCH /api/images/{id}/comments/{commentId}` | 편집 |
| `DELETE /api/images/{id}/comments/{commentId}` | 삭제 |

- 모든 프로젝트 멤버가 코멘트 조회/작성 가능
- 편집/삭제는 작성자 본인만

## 테스트 계획

- 코멘트 작성 → 스레드에 즉시 표시
- @멘션 자동완성 → 프로젝트 멤버 표시
- 본인 코멘트 편집/삭제 → 반영
- 타인 코멘트 편집/삭제 시도 → 차단
