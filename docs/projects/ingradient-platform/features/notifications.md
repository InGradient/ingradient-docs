# 알림 (Notifications)

## 기능 개요

학습 완료/실패, 코멘트 @멘션 등 주요 이벤트를 사용자에게 알리는 시스템이다.

## 문제 정의

학습이 수시간 걸리거나, 다른 팀원이 코멘트로 의견을 요청한 경우 사용자가 능동적으로 확인하러 가지 않아도 알 수 있어야 한다.

## 범위 / 비범위

- **포함**: 학습 완료/실패 알림, @멘션 알림, 읽음 처리, 보관
- **제외**: 이메일/푸시 알림 → 향후, 실시간 WebSocket → 현재 폴링 방식

## 알림 종류

| 종류 | 트리거 | 내용 |
|------|--------|------|
| 학습 완료 | Training job → Completed | "모델 {name} 학습이 완료되었습니다" |
| 학습 실패 | Training job → Failed | "모델 {name} 학습이 실패했습니다" |
| 코멘트 멘션 | 코멘트에서 `@사용자` | "{작성자}님이 코멘트에서 회원님을 멘션했습니다" |

## 동작 방식

- 5~10초 간격 폴링으로 새 알림 확인
- 프로젝트별 또는 전역 범위 조회
- 읽음 처리 (mark as seen)
- 보관 (archive) — 알림 목록에서 숨김

## 데이터 / API / 권한 영향

| API | 용도 |
|-----|------|
| `GET /api/notices` | 알림 목록 조회 |
| `POST /api/notices/{id}/seen` | 읽음 처리 |
| `POST /api/notices/{id}/archive` | 보관 |

## 테스트 계획

- 학습 완료 시 알림 생성 확인
- @멘션 코멘트 작성 시 대상 사용자에게 알림
- 읽음 처리 → 미읽음 카운트 감소
- 보관 → 알림 목록에서 제거
