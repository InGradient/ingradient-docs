# 계정 관리 (Account Management)

## 기능 개요

사용자 프로필 편집, 비밀번호 변경, 계정 삭제를 처리하는 기능이다.

## 문제 정의

사용자가 자신의 계정 정보를 직접 관리할 수 있어야 하며, 계정 삭제 시 프로젝트 소유권 등 영향 범위를 사전에 확인할 수 있어야 한다.

## 범위 / 비범위

- **포함**: 프로필 편집, 비밀번호 변경, 계정 삭제 (영향 미리보기)
- **제외**: 계정 생성 → Auth (회원가입), 조직 수준 멤버 관리 → Admin

## 사용자 시나리오

### 프로필 편집

- Settings 모달 → Account 탭
- 이름, 이메일 변경 후 저장

### 비밀번호 변경

- 현재 비밀번호 확인 → 새 비밀번호 입력 → 저장
- auth-service의 비밀번호 정책 적용

### 계정 삭제

- **기본 흐름**:
  1. "계정 삭제" 클릭 → 미리보기 대화상자
  2. 영향 받는 프로젝트 목록 표시 (소유 프로젝트)
  3. 소유 프로젝트가 있으면 소유권 이전 옵션 제공
  4. 확인 입력 → 삭제 실행
- **예외 흐름**:
  - 소유 프로젝트의 유일한 멤버인 경우 → 프로젝트도 삭제됨 경고

## UI / UX 방향

- Settings 모달의 Account 탭
- 삭제 버튼은 위험 색상(빨간색) + 확인 대화상자
- 미리보기에서 영향 범위를 명확히 표시

## 테스트 계획

- 프로필 변경 → 저장 → 재로딩 시 유지
- 비밀번호 변경 → 새 비밀번호로 재로그인
- 계정 삭제 미리보기 → 프로젝트 영향 표시
- 소유권 이전 후 삭제 → 프로젝트 유지 확인
