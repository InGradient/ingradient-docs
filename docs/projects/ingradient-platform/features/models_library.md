# 모델 라이브러리 (Models Library)

## 기능 개요

사전 학습된 AI 모델을 탐색, 다운로드하고 플랫폼 작업(자동 라벨링, 분류, 임베딩 등)에 배정하는 기능이다.

## 문제 정의

자동 라벨링이나 추론에 사용할 모델을 프로젝트에 쉽게 배정하고 관리할 수 있어야 한다. 모델 크기와 속도의 트레이드오프를 사용자가 이해하고 선택할 수 있어야 한다.

## 범위 / 비범위

- **포함**: 모델 브라우징, 다운로드, 삭제, 작업별 모델 배정
- **제외**: 모델 학습 → Training, 모델 학습 파일 저장 → ingradient-ai

## 사용자 시나리오

### 모델 다운로드

- **기본 흐름**:
  1. `/models` → Library 탭
  2. 카테고리별 모델 탐색 (크기, 정확도, 속도 정보 표시)
  3. 원하는 모델 "다운로드" 클릭
  4. 진행률 표시 → 완료
- **예외 흐름**:
  - 다운로드 실패 → 재시도 가능
  - 디스크 부족 → 에러 표시

### 모델 작업 배정

- **기본 흐름**:
  1. `/models` → Assignments 탭
  2. 작업별로 모델 선택:
     - `object_detection` — 자동 바운딩 박스 라벨링
     - `classification` — 자동 분류 라벨링
     - `umap_embedding` — 차원 축소 시각화
  3. 현재 배정 상태 확인, 변경 시 저장

## UI / UX 방향

| 탭 | 구성 |
|----|------|
| Library | 카테고리별 모델 카드 (이름, 크기, 속도, 정확도), 다운로드/삭제 버튼, 진행률 |
| Assignments | 작업 목록 (object_detection, classification, ...), 각 작업에 모델 드롭다운 |

## 데이터 / API / 권한 영향

| API | 용도 |
|-----|------|
| `GET /api/models/library` | 사용 가능한 모델 목록 |
| `POST /api/models/{id}/download` | 다운로드 시작 |
| `DELETE /api/models/{id}` | 다운로드된 모델 삭제 |
| `GET /api/models/assignments` | 현재 배정 상태 |
| `PATCH /api/models/assignments` | 배정 변경 |

## 테스트 계획

- 모델 다운로드 → 진행률 → 완료 → Library에 표시
- 다운로드된 모델 삭제 → 목록에서 제거
- 모델 배정 → Gallery에서 자동 감지 시 해당 모델 사용
